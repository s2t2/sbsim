# Smart Control Documentation

This directory contains the source files for the Smart Control project documentation.
The documentation is built using [Sphinx](https://www.sphinx-doc.org/) with the `sphinx-rtd-theme`.
API documentation is automatically generated from docstrings in the `smart_control` package using `sphinx-autoapi`.

## Building the Documentation Locally

1.  **Ensure dependencies are installed:**
    If you haven't already, install the project dependencies, including those required for documentation:
    ```bash
    poetry install --with dev
    ```

2.  **Build the HTML documentation:**
    Navigate to the `docs` directory and use the `sphinx-build` command (or `make html` if a Makefile exists).
    It's generally recommended to run sphinx-build directly:
    ```bash
    cd docs
    poetry run sphinx-build -b html . _build/html
    ```
    If a `Makefile` was generated by `sphinx-quickstart` and you prefer to use it:
    ```bash
    cd docs
    poetry run make html
    ```
    This will generate the HTML files in `docs/_build/html/`.

3.  **View the documentation:**
    Open `docs/_build/html/index.html` in your web browser.

## Maintaining the Documentation

-   **Docstrings:** Most of the API documentation is generated from docstrings in the Python code (`../smart_control` directory). Ensure that all public modules, classes, functions, and methods have clear and comprehensive docstrings following a consistent style (e.g., Google style, which `napoleon` extension can parse).
-   **Sphinx Configuration:** The main configuration file is `docs/conf.py`. This file controls Sphinx extensions, project settings, and the HTML theme.
-   **AutoAPI Configuration:** `sphinx-autoapi` is configured in `docs/conf.py` via the `autoapi_type`, `autoapi_dirs`, `autoapi_options`, and `autoapi_ignore` settings.
-   **Index Page:** The main entry point for the documentation is `docs/index.rst`. You can modify this to change the structure of the main page.
-   **Adding New Markdown Files:** If you want to add new documentation pages written in Markdown, create `.md` files and link them in a `toctree` in `index.rst` or other relevant `.rst` files. Ensure `myst_parser` is enabled in `conf.py`.

## GitHub Pages Deployment

A GitHub Actions workflow is set up in `.github/workflows/docs.yml` to automatically build and deploy the documentation to GitHub Pages whenever changes are merged into the main branch.
